hexo+GitHub简单搭建流程

发表于 2019-04-18 | 更新于 2019-04-19

环境配置

  • 工具：Git
  • 环境：node（下载左侧稳定版即可）

安装过程全程下一步

在本地电脑安装hexo

1.找个位置新建文件夹，右击该文件夹使用Git bush打开

2.输入命令

npm install -g hexo-cli

3.参考下图，安装完毕后可输入

hexo -v

检查hexo版本以及是否安装好。（此步可略过）
上述步骤完毕后文件夹依然为空，进行下一步初始化之后文件夹内才会出现文件

4.输入命令初始化hexo，下述代码中”blog”为即将新建的文件夹名，可随意填写

hexo init blog

5.输入 

cd blog

其中”blog”为你的新文件夹名
回车

6.输入

npm install

回车

7.安装完成后输入

hexo g

回车

8.输入

hexo s

回车

完成后用浏览器打开localhost:4000即可看到预览页面，在Gitbush下按CTRL+c即可关闭预览

建立GitHub库

GitHub网址
注册并登陆后首页↓

此处填写为你的GitHub用户名 + .github.io
如图我的二级域名即为 Jaa233.github.io
其他选项不用动，建库

上传SSH公钥

9.回到Git bush中，输入

git config --global user.name "你的GitHub用户名"

如git config --global user.name "Jaa233"
回车
10.输入

git config --global user.email "你在GitHub上绑定的邮箱"

如git config --global user.email "123@qq.com"
回车
11.

ssh-keygen -t rsa -C "你在GitHub上绑定的邮箱"

如ssh-keygen -t rsa -C "123@qq.com"
回车

注：如图9步和10步程序没有可见反应，可用以下两条命令（红色箭头）检查信息是否正确（红色横线标记）

git config user.name
git config user.email

第11步需要手动按三次”回车enter”，不需要输入任何东西，直接按回车（蓝色箭头所指信息出现时）


命令执行完毕后会显示已生成了.ssh文件夹，在电脑中找到这个文件夹，生成位置基本一样，在你的Gitbush中找到图中绿色横线标记的位置即为你的存储路径，我的电脑用户名为Jiang，路径即为

C:\Users\Jiang\.ssh

id_rsa是私人秘钥，不能给别人看
id_rsa.pub是公共秘钥，可以给别人看。此处我们用到的就是公钥

把公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

进入GitHub
打开setting
找到SSH and GPG keys
点击New SSH key
把你的id_rsa.pub里面的信息复制进去。
点击Add ssh key

在Git bush中用 

ssh -T git@github.io

查看是否成功

将hexo部署到GitHub

打开根目录下_config.yml文件（用记事本即可），翻到页面底部，将

deploy:
  type:

修改为

deploy:
  type: git
  repo: git@github.com:你的GitHub用户名/你的GitHub用户名.github.io.git
  branch: master

注意冒号后应有空格，否则会报错
repo后内容即为GitHub库的ssh


然后安装deploy-git，即为部署的命令

npm install hexo-deployer-git --save

图为安装成功

然后

hexo clean

回车

hexo generate

回车

hexo deploy

hexo clean清除缓存
hexo generate重新生成代码，可以用hexo g缩写
hexo deploy部署，可用hexo d缩写

注：deploy时可能会要求输入username和password
安装部署命令时可能会报错需要eslint4.12.1版本，可用

npm install eslint@4.x babel-eslint@8 --save-dev
